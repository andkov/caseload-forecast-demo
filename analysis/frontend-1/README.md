# Frontend-1: Design Workspace

This folder captures the design instructions, editorial specifications, and evolution of implementation decisions for the **`_frontend-1/`** website workspace.

---

## Purpose

This is the **thinking space** for frontend-1. Editorial intent is documented here with inline reasoning and revision history. The execution space is `_frontend-1/`.

## Contents

| File | Purpose |
|------|---------|
| `initial.prompt.md` | Human editorial brief — annotated source of truth for what should appear on the website, with inline rationale for every choice |
| `frontend-1-map.md` | Historical visual map of the pipeline (created in first design iteration; superseded by `.github/publishing-orchestra-2.md`) |

## System Design Reference

For the current design of the Publishing Orchestra system, see:

- `.github/publishing-orchestra-2.md` — Comprehensive design guide (current, v2)

## How to Use

1. Open `initial.prompt.md` to review and revise editorial intent for this frontend.
2. Copy the revised content (without inline comments) to `_frontend-1/editor.prompt.md`.
3. Invoke `@publishing-orchestrator` to run the pipeline.
